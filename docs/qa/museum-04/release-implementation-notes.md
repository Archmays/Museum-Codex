# MUSEUM-04 release implementation notes

The formal release is generated by `museum_pipeline.art.public_release` from immutable MUSEUM-03B graph inputs and the validated MUSEUM-03C bundle. The generator writes into a hidden staging directory, performs full validation, and atomically installs the directory only after success; it refuses to overwrite an installed release with different bytes.

## Exact projection boundary

Validation does not trust a mutually consistent public projection. It reconstructs canonical expected values and compares:

- 12 artist, 44 artwork, 31 context, and 36 relation records;
- every C-level relation, supporting artwork, Claim, Evidence, Source, limitation, and non-causal statement;
- all 31 source-provenance media parents and 242 derivative children;
- source IDs/URLs, content hashes, derivation hashes, rights status, license descriptor, verified dates, permission flags, attribution, source-rule binding, notices, and every withdrawal row;
- manifest paths, byte lengths, SHA-256 values, typed IDs, schema dispatch, and physical JPEG/WebP magic bytes;
- zero media for all 13 metadata-only or blocked artworks and zero external runtime delivery.

Synchronized tampering of the projection, claims envelope, attribution, notice, and manifest still fails because the expected values come from M03B/M03C. Malformed nested manifest/media shapes return structured failure codes instead of terminating validation.
