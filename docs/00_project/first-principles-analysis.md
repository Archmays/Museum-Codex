# 第一性原理分析

## 用户为什么需要数字博物馆

互联网上缺少的不是信息，而是可信的选择、语境和可追问路径。用户需要知道对象为何值得放在一起、某条线是事实还是比较、图像能否合法复用、结论来自哪里，以及结论改变后如何追踪。Museum 的价值函数因此是：

> 可理解的探索 × 可复核的证据 × 可持续的发布，而不是条目数量 × 视觉冲击。

## 博物馆的最小完备单元

一个可公开展陈单元至少包含：对象身份、策展语境、可验证陈述、证据与来源、权利结论、版本和可访问表达。缺少其中任一项时，它只能停留在候选区或开发区。

知识底座区分三种真值承担者：

1. 实体回答“我们在谈什么”；
2. Claim 回答“我们声称什么”；
3. Evidence 与 Source 回答“凭什么、在哪里、何时核验”。

关系是受控 Claim 的可视化投影，而不是从几何距离倒推出的历史事实。

## 防止视觉语法制造事实

- 图谱布局只解决遮挡和导航；坐标与距离没有艺术价值含义，界面必须声明这一点。
- `historical_relationship_strength`、`evidence_confidence`、`computational_similarity`、`curatorial_relevance` 分字段、分图例、分筛选。
- 颜色编码关系类型，同时配合线型、图标、文本和可访问名称。
- A 级直接证据可用实线；B 级历史语境用虚线；C 级算法或策展比较用点线并默认可关闭。
- 边粗细只能映射已命名度量，不能把“看起来强”当成“确定为影响”。
- 算法结果需要模型/特征/数据集版本、适用范围和误差说明；不能通过工作流改名为 `explicitly_influenced_by`。

## 多分馆为何能共享底层

所有学科都需要稳定身份、时间、地点、陈述、证据、来源、媒体权利和版本。但“艺术家影响”与“捕食关系”的证据、方向、时效和风险不同。因此采用小型通用 schema 加分馆扩展：共享可治理机制，不共享未经证明的学科假设。

例如，一幅作品的传统归属可以并存多个 Claim；物种分布也可并存观测、推定范围和评估结论。两者共享冲突保留与证据链，但使用不同谓词、审核者和发布门槛。

## 外部 API 失效时为什么仍可用

权威 API 是采集界面，不是公共站点依赖。每次采集保存原始响应、请求参数、抓取时间和内容哈希；清洗、审核后生成带 manifest 的静态 JSON/GeoJSON、搜索索引与获准媒体衍生物。Pages 只读取同一 release 目录。上游中断只影响下一次更新，不影响当前版本；回滚是切换 release 指针并保留撤回公告。

## 反例与设计答案

| 失败方式 | 原因 | 结构性防线 |
|---|---|---|
| 两幅画像相似就连“影响”线 | 把相关性变成因果 | C 级专用语义、schema 条件、测试 fixture |
| API 返回图片即公开托管 | 混淆接口与权利 | 元数据/媒体许可分字段、发布门禁 |
| 修正出生日期时覆盖旧值 | 实体字段成为唯一事实源 | 版本化 Claim、废弃/取代关系、审计历史 |
| 一次观测变成物种稳定分布 | 混淆记录与推断 | observation 与 distribution Claim 分型 |
| 一次加载完整大图谱 | 展陈逻辑被数据规模支配 | 局部子图、分层筛选、静态分片和渐进加载 |
