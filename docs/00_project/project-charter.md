# MUSEUM-00 项目章程

## 使命

建立一个可信、可解释、可持续扩展的数字博物馆：用户通过展陈、关系、时空与路径形成问题并追踪证据，而不只是检索孤立条目。首馆为美术馆，通用底层同时服务生物、音乐、游戏、文明、武器与科学分馆。

## 服务对象与需要

| 人群 | 核心需要 | 产品承诺 |
|---|---|---|
| 儿童与家庭 | 从观察和问题开始，避免信息压垮 | 分层导览、明确互动目标、非颜色单一编码 |
| 普通公众 | 理解“为什么有关联” | 每条关系可展开解释、证据与不确定性 |
| 深度学习者 | 复查来源、比较争议、追踪版本 | Claim–Evidence–Source、引用定位、版本与撤回日志 |
| 策展与研究协作者 | 安全地加入候选而不污染正式数据 | 分区流水线、人工审核、权利门禁、可回滚发布 |

## 范围

`MUSEUM-00` 交付愿景、领域和知识图谱模型、证据/来源/权利治理、来源核验、数据发布流程、MVP 与研究边界、ADR、JSON Schema、fixtures、验证器、测试、路线图、风险与阶段报告。

本阶段不交付正式网页、交互、采集批次、媒体、艺术家名单、Pages 工作流或运行时应用。结构占位不能被称为产品完成。

## MUSEUM-02A 章程修订

当前总架构登记七个分馆。第七分馆为 `arms`，中文公开名称“武器博物馆”，英文公开名称 “Museum of Arms & Armor”。它在门户中处于 `planned / portal-visible / data-contract-not-yet-implemented`：入口卡片可见但不可导航，不含正式器物、媒体、数据 release 或分馆 schema。

本修订只登记新的分馆边界与安全策展责任，不回写 `MUSEUM-00` 当时的交付状态，不改变 `MUSEUM-03` 的美术馆目标或首批 12 位艺术家范围。武器馆正式研究须由用户另行授权进入 `MUSEUM-ARMS-00`。

## 原则

1. **展陈优先于堆积**：藏品进入可解释语境；搜索只是入口。
2. **陈述可追溯**：事实不是字段装饰，而是有状态、有证据、有反证的记录。
3. **不确定性可见**：传统归属、争议、算法结果和策展观点不伪装成事实。
4. **权利逐资产**：开放 API 不推导出开放图片。
5. **静态优先**：发布快照在上游 API 中断时仍可浏览；客户端断网浏览只有在未来完成预缓存、版本更新与撤回传播设计后才承诺。
6. **分馆自治、底层共享**：共享身份、证据、地点、时间、权利和版本；学科语义由扩展 schema 负责。
7. **可撤回**：错误修复留下版本和取代关系，不改写历史。

## 治理角色

- 数据维护者：采集、标准化、生成候选，不批准事实。
- 学科审核者：确认身份、陈述、证据等级和解释。
- 权利审核者：确认资产级许可与发布范围。
- 发布维护者：运行门禁、签发数据集、执行撤回；不能绕过失败检查。
- 产品维护者：将证据语义准确映射到界面，不重新解释评分。

同一人可以兼任角色，但每个动作仍需独立记录。正式发布至少需要学科审核与权利/发布门禁两类结论。

## 成功标准

章程所列机器规则均有 schema、fixture 或验证器支撑；来源登记含当前官方 URL 与分离许可结论；所有无效 fixture 被拦截；A–E 审查修复后通过；仓库只含治理底座；Git 状态可复核。
