# 产品愿景与边界

## 产品定义

Museum 是“可验证知识驱动的数字展陈系统”。它把藏品、人物、地点、时间、事件和概念编入策展问题；关系图、地图、时间轴和 AB 路径是论证界面，不是装饰性可视化。

### 与相邻产品的区别

| 类型 | 优化目标 | Museum 的不同点 |
|---|---|---|
| 百科 | 条目覆盖和快速释义 | 以展览和问题组织，保留冲突陈述及证据链 |
| 搜索引擎 | 找到外部页面 | 只在审核快照中探索，并解释排序/关联 |
| 图片库 | 查找和观看媒体 | 媒体服务于藏品解释，权利逐资产可见 |
| 社交推荐 | 增加停留和相似推荐 | 算法相似默认可关闭，永不冒充历史影响 |
| 传统馆藏系统 | 管理内部对象记录 | 面向公众探索，同时保留来源、版本与撤回语义 |

## “藏品”“关系”“陈述”的边界

- **藏品**：由某分馆策展语境承载、具有稳定身份或受控归属、可引用来源且具展示权利结论的研究对象。`artwork`、标本或经审定的数字记录可成为藏品；任意网页、无来源图片和算法生成对象不能自动成为藏品。
- **关系**：两个实体之间具有命名语义、方向、范围、证据等级和展示政策的连接。共享标签不是默认关系；动态计算结果需标记计算方法与版本。
- **Claim**：可被支持、反驳、争议、废弃或撤回的原子命题。实体字段可以是经过验证 Claim 的投影，但不能成为不可追溯的第二事实源。
- **策展说明**：对已明确事实与比较框架的解释，不增加未经证实的历史主张。

## 体验层级

1. 观察：先看作品、对象、时间或地点。
2. 连接：展开少量相关关系并解释连接缘由。
3. 比较：通过 AB 路径、对照展台或专题展看到差异与共同点。
4. 证据：按需查看陈述、来源、权利和争议。
5. 深挖：筛选、沿路径探索并引用稳定版本。

儿童、普通公众、深度用户共享同一事实底座，只改变解释密度、任务提示和默认可视范围。

## 明确非目标

- 不建立“最伟大艺术家”排名或用节点大小表达地位。
- 不以全网实时聚合、无审核 UGC、生成式替代藏品或沉浸技术本身为目标。
- 不保证每件作品拥有每种互动；互动只在素材、权利、学科价值和无障碍条件满足时启用。
- 不在浏览器运行时拼接多个权威 API 作为正式事实层。

## 共享与自治

门户共享身份、检索、时间、地点、证据、来源、媒体、权利、国际化、无障碍和发布版本。分馆分别拥有学科实体、关系词表、证据解释、互动适用条件与审核角色；不得为了统一 schema 抹平“艺术归属”和“物种观察”等差异。
