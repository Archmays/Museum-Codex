# 分馆策略

## 门户与分馆契约

门户提供分馆入口、统一搜索、全局时间/地点索引、未来收藏与探索历史接口、语言切换、键盘导航、可访问偏好和数据版本提示。门户不重新定义分馆事实。

分馆标识固定为：`art`、`biology`、`music`、`games`、`civilization`、`arms`、`science`。当前只有 `art` 具有可进入的序厅；`biology` 只建立研究模型；`arms` 为 `planned / portal-visible / data-contract-not-yet-implemented`；其余只保留接口。

`portal-visible` 只允许显示不可点击的筹备卡片和公众可理解的状态文字，不等于分馆原型、数据发布或正式入口。当前 `arms` 不得新增路由、器物记录、媒体、adapter 或 live probe。

每个分馆扩展必须声明：

- 自有实体与关系词表及 schema 版本；
- 依赖的 common schema 版本范围；
- 学科证据门槛、审核角色和不确定性词汇；
- 可用的展陈类型和无障碍替代；
- 权利/伦理/安全附加门槛；
- 数据导入、撤回和跨馆链接策略。

## 共享能力与不可共享假设

| 共享能力 | 分馆必须自定义 |
|---|---|
| 稳定 ID、labels、外部 ID、地点、时间 | 学科身份规则与精度 |
| Claim/Evidence/Source、审核状态 | 可接受证据、谓词与争议表达 |
| 媒体权利、归属、许可期限 | 标本、作品、声音等特定权利/伦理 |
| dataset release、hash、withdrawal | 分馆发布门槛和更新频率 |
| 搜索、i18n、无障碍外壳 | 展陈语法、互动任务和叙事层级 |

## 跨馆连接

跨馆连接必须通过稳定实体或明确的 cross-branch Claim 建立。例如颜料材料可连接 art 与 science，栖息地绘画可连接 art 与 biology；连接不能仅因同词标签自动产生。每条跨馆关系需指定主审分馆、协审分馆、展示上下文和冲突解决责任。

## 新分馆准入

新分馆只有在来源清单、身份规则、证据政策、权利风险、至少一组 schema/fixtures、审核负责人和静态发布方案完成后才能进入原型阶段。没有最低可浏览、可追溯数据集时，不得提供可导航的分馆入口；在跨馆架构修订中可以登记不可点击的 `portal-visible` 筹备卡片，但必须同时标明数据契约未实现并保持 canonical dispatch fail-closed。

武器博物馆还必须先完成独立的安全、伦理、年龄适宜性和防操作化门禁。其公共价值在于从器物、甲胄、材料、制度、象征、来源、保存与人类代价理解历史，而不是购买、性能排名、现实使用训练或暴力娱乐化。
