# MUSEUM-01 入场门槛

## 必须满足

- `phase-museum-00-report.md` 为 `completed`、`validation_status=pass`、`governance_foundation_ready=true`；
- common/art/biology schema、valid/invalid fixtures、三个验证器和 tests 在干净 checkout 通过；
- 固定依赖能在受支持 Python 的新虚拟环境安装；物理 release fixture 的 canonical schema 分派、精确文件集合、路径、字节、SHA-256、类型化 ID、引用闭包、Source rule/决策快照、自托管媒体、状态、Tier、来源/媒体权利、notices 与 attribution 实际内容校验通过；
- source registry 有官方 URL、核验日期、API/Key/限制、元数据与媒体许可分列；
- A–E 审查已修复 substantive findings；
- Git commit 可定位，工作区范围清楚，无密钥/大文件/未审媒体；
- 用户明确授权开始 `MUSEUM-01`。

## 启用公开 Pages 前另需满足

- 用户确认仓库/站点可见性、代码许可证和原创内容许可范围；
- Actions 最小权限、固定依赖、base path、路由和回滚 dry run；
- 站点只有明确标为占位/非正式的数据，或引用通过 release gate 的内容；
- accessibility/low-bandwidth 错误状态有最小实现。

## MUSEUM-01 输入包

本阶段全部架构/治理文件、schema/fixtures/validator、固定 source registry 版本、open decisions、risk register、Git 状态与 MUSEUM-00 commit。MUSEUM-01 不需要也不得提前接收首批 12 人正式名单或批量媒体。

## 退出门槛摘要

React/TypeScript/Vite 最小壳、门户与分馆入口占位、i18n/a11y 基线、静态 release loader、Actions 验证/构建/Pages dry run、路由深链和回滚验证；不包含艺术星海、地图、AB 路径或展厅。
