# 美术馆 MVP 范围

## 目标

用一个规模可控、证据完整的首展证明：公众可以从艺术家/作品出发，理解关系为何成立，沿可解释路径比较，并随时回看来源和权利。MVP 不是“艺术史大全”。

## 候选规模（MUSEUM-03 才确定名单）

- 12 位确认去世且身份可解析的艺术家；
- 每位约 4 件具有对象级公开展示权利的作品（总计约 48 件）；
- 50–80 条可解释、审核后的关系；
- 中英文 preferred names/aliases，完整来源与逐资产许可；
- 移动端与 WCAG 2.2 AA 基线。

数量是预算，不是质量门槛；若来源或权利不达标，应缩小内容而不是用无来源或生成替代图补齐。

## 必备体验（未来阶段）

艺术星海局部探索、艺术家简介面板、艺术家数字展厅、作品详情、双画对照、至少一类 AB 路径、关系解释、来源/许可视图、关系/等级/时间/地区筛选和关闭算法相似。

## 内容门槛

- 艺术家死亡与身份 Claim 达 publishable；匿名/工作室归属使用专门表达。
- 作品对象、创作者归属、日期/范围、持有机构和媒体权利可追溯。
- 关系至少 reviewed 且适配 A/B/C 文案；公开边没有撤回 Claim。
- 每个媒体通过对象级 rights gate，提供归属和无障碍文本。
- 展签事实全部链接 Evidence；AI 只作为已审核草稿来源记录。

## 不在 MVP 的承诺

完整艺术史覆盖、所有作品均有全部互动、艺术价值排名、实时外部 API、社交账户、UGC、沉浸式 3D 展厅、AI 艺术家替代肖像或未经证据的推荐关系。

## 成功指标（在实现阶段定义数值目标）

- 用户可从任一可见关系一跳打开解释/证据；
- 键盘和文本列表完成与图谱等价的邻域与 AB 路径探索；
- 上游 API 断开时固定 release 仍可浏览；
- 权利未知、开发资产、withdrawn 记录零进入公开构建；
- 移动目标设备在性能预算内加载局部图和响应式图片；
- 用户测试能区分“历史关系”“策展比较”“算法相似”。
