# 内容深度分层

## Gallery

Gallery tier 表示身份、已故状态、来源和作品容量足以支持未来完整艺术家展厅，原则上可组织 8–15 件作品。MUSEUM-09A 注册 125 位 Gallery target。

Gallery 不代表艺术价值更高，也不表示已经取得媒体许可。没有可用图片时，后续阶段仍可选择 metadata-first；逐对象媒体权利、图像质量和展示编排留给获授权的执行批次。

## Collection

Collection tier 表示艺术家可进入未来索引与作品目录，但不要求立即具备完整展厅深度。每位艺术家必须有稳定身份、确认已故、非 Wikidata 正式来源、Claim → Evidence → Source 闭合和至少三件目标作品；原则上配置 3–10 件。MUSEUM-09A 注册 375 位 Collection target。

## Discovery-only

Discovery-only 只供内部研究，包括未被本轮 target/reserve 选中的合格人物，以及身份冲突、死亡状态未闭合、作品不足、匿名、工作室、群体、文化或传统归属、重复实体和来源不足的记录。它们保留状态与 reason code，不公开，也不静默删除。

不满足个人、已故、正式来源或三件作品硬门禁的记录不能因为覆盖缺口而升级。`reserve` 另有固定替补顺序，只能在目标人物 hard gate 失败时按顺序使用。

## 本阶段分布与变更规则

| Tier | Target 数 | 本阶段含义 |
|---|---:|---|
| Gallery | 125 | 未来 8–15 件深度展厅候选 |
| Collection | 375 | 未来 3–10 件目录候选 |
| Discovery-only | 0 target | 仅内部发现、拒绝、阻塞或未选记录 |

Tier 分配由可验证作品容量、来源闭合和批次可执行性决定，不使用重要性、流行度、市场价值或审美分数。M09A 只注册 tier；具体展厅写作、媒体选取、关系研究和公开发布必须由后续获授权阶段完成。
