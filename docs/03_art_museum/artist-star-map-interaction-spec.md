# 艺术星海交互规格（规划）

## 探索目标

星海帮助用户提出“这条关系是什么、证据在哪里、还有哪些替代路径”，不是展示最密集或最有地位的人。图谱只是固定 release 的一个视图。

## 初始状态

- 不加载/显示全部关系线；节点默认等大，显示少量代表性标签。
- 提供搜索、列表入口、关系类型/等级/时间/地区筛选和“算法相似”总开关（默认关）。
- 图例声明颜色、线型、透明度、线宽与距离的含义/非含义。

## 选择艺术家后

1. 以所选节点为焦点，展开一跳局部关系；相关节点高亮，无关节点降低亮度但不消失。
2. 边以类型颜色、A/B/C 线型和可访问标签显示；点击/键盘选择打开关系解释。
3. 解释面板显示端点、关系类型、范围、等级、置信、策展说明、Claim/Evidence/Source、争议和 release。
4. 用户可固定节点、返回全局、加入 AB 端点或关闭某类边。

## 状态与 URL

可分享状态只包含 release、焦点 ID、过滤器和视图模式，不嵌入未经审核文本。无效/撤回 ID 显示明确说明和可用替代。布局坐标可构建期预计算以降低移动 CPU；坐标不是事实字段。

## 移动与无障碍

移动端默认更小邻域、底部详情面板和更少标签；不要求悬停。键盘可在节点/边/筛选间移动并有可见焦点；屏幕阅读器使用同步实体列表与关系表，状态变化通过 live region 摘要。`prefers-reduced-motion` 禁用持续物理动画。

## 性能门槛（MUSEUM-04 定量）

必须用目标低/中端移动设备验证首个局部图加载、选择响应、内存和帧率；分别以 1k/10k/50k 合成图做边界基准。若不达标，优先分片、抽稀、预计算和 Web Worker，不减少证据字段。

## 禁止语义

不显示“影响力大小”节点；不以中心位置表示地位；不默认把 C 级边混入历史路径；不把过滤后的缺失解释成不存在关系。
