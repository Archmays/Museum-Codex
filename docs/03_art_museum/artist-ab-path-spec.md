# 艺术家 AB 路径规格（规划）

## 问题

AB 路径回答“在当前审核数据中，可以通过哪些可解释关系从 A 走到 B”，而不是证明唯一因果链或最短历史影响链。

## 输入与约束

- 起点/终点 stable ID 和固定 dataset release；
- 允许关系类型、A/B/C、时间/地区、公开状态；
- 是否允许有向边逆行显示；默认遵守语义方向；
- `include_algorithmic=false` 默认值；
- 最大 hops、候选路径数、认知负担预算。

所有边先通过 `public_display`、review status、withdrawal、rights/visibility 和 filter，再参与计算。

## 排序

默认优先：更少跳数、较高证据等级、每边有清楚解释、时间语境连贯、关系类型不过度重复。四类分数不能压成一个看似客观的“影响力”权重。具体排序函数和 tie-break 必须带算法版本并在界面解释。

建议提供最多 3 条有意义的替代路径：历史路径（A/B）、语境路径（允许 B）、比较路径（明确启用 C）。若没有路径，说明当前 release/过滤器下无可展示路径，不声称现实中没有关系。

## 输出

路径记录端点、有序节点/边、每边关系解释、Claim/Evidence 入口、过滤器、算法版本、生成时间、release 和免责声明。用户可以逐边展开并比较替代路径。

## QA

- C/算法边默认排除，显式启用后样式和文案仍分离。
- 有向关系不会因展示方便反转事实；反向浏览明确标示。
- withdrawn/deprecated/unreviewed 边不进入公开路径。
- 时间不相容路径得到提示或降级，不被称为接触链。
- 键盘/文本模式可完整完成端点选择、路径切换和证据查看。
