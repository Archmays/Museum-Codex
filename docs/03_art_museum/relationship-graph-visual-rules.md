# 关系图视觉规则

## 规范映射

| 语义 | 映射 | 辅助表达 |
|---|---|---|
| relationship type | 受控色板 | 图标/缩写/完整文字 |
| A 直接事实 | 实线 | `A 直接证据` 标签 |
| B 历史语境 | 长虚线 | `B 历史语境` 标签 |
| C 比较/算法 | 点线 | `C 策展比较` 或 `C 算法相似` 标签 |
| evidence confidence | 三档透明度 | low/medium/high + 原始值/说明 |
| historical strength | 有定义时 3 档线宽 | 指标名、单位、null 说明 |
| computational similarity | 独立数值/百分位 | 方法与 release；不映射线宽 |
| curatorial relevance | 当前展览的排序/外光环 | `本展相关`；不改变证据样式 |

色板在浅/深背景及常见色觉缺陷模拟下满足对比；不得只依靠红/绿。选中状态增加轮廓、形状或图标，不只改颜色。

## 节点

默认等大。若某视图临时按局部连接数调整大小，必须可关闭、显示图例，且文案明确“不是艺术地位”。身份不确定、匿名/群体实体用不同形状和文本标签，不伪装个人。

## 布局

布局仅减少遮挡和支持导航。界面固定说明：空间距离、中心位置和聚类不是客观艺术价值、亲密程度或影响证明。构建期布局版本写入 release；布局变化不产生 Claim 变化。

## 信息过载控制

默认一跳、按需扩展；边标签在焦点/键盘选择时出现；控制同屏边数并保留“还有 N 条”入口；筛选结果显示隐藏原因。用户可单独关闭算法边或 C 级边。

## 退化模式

WebGL 不可用、减少数据/运动偏好或辅助技术模式下，使用排序列表/关系表完成同一任务：选择实体、筛选、打开解释、设 AB 端点。导出/截图必须包含图例、release 和“距离无事实含义”的说明。

## MUSEUM-04 正式发布 profile

- 本 release 的关系等级精确为 `A=0 / B=0 / C=36`；C 全部表示人工审核的策展比较，算法相似与历史因果关系均为 0。
- 36 条 C 边使用 Sigma 3.0.3 的本地 custom edge program 绘制真实点线，同时保留类型图标、`C｜策展比较` 文字、端点和“并不意味着什么”的解释。不得降级为 solid-only，也不得使用与 pan/zoom 失步的 SVG 覆盖层。
- 12 个艺术家节点默认且始终等大；选中状态使用轮廓/halo/文字冗余。MUSEUM-04 不提供按度数缩放、排名或中心艺术家模式。
- Graph、Artist list、Relationship table 是共享同一状态与 URL 的三种等价正式体验。列表和表格不是次等 fallback；本阶段不设置 AB 端点，也不提供 AB 路径。
- forced colors、低带宽或 WebGL 失败时自动进入列表并公告原因，保留搜索、筛选、焦点、关系解释和来源；除非访客主动切换，低带宽模式不初始化 Sigma。
- 初始图只显示 12 个节点和有限 labels，不显示边；聚焦艺术家后才显示一跳关系。布局由 release 固定 seed/version/hash，运行时无随机或持续力导向。
